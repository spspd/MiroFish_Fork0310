# MiroFish PM Milestone Table

`commit_archieve`를 기준으로 역추정한 PM용 마일스톤 표다.
실제 운영된 공식 PM 문서가 아니라, 커밋 흐름을 기준으로 재구성한 관리용 초안이다.

## PM 요약

| Milestone | 기간 | PM 관점 목표 | 상태 추정 |
| --- | --- | --- | --- |
| M1. 그래프 MVP | 2025-11-26 ~ 2025-11-29 | 텍스트를 그래프로 전환하는 최소 백엔드 파이프라인 확보 | 완료 |
| M2. 시뮬레이션 엔진 결합 | 2025-12-01 ~ 2025-12-09 | 그래프 기반 에이전트 시뮬레이션 실행 가능 상태 확보 | 완료 |
| M3. 리포트 자동화 | 2025-12-09 | 시뮬레이션 결과의 자동 보고서화 구현 | 완료 |
| M4. 제품형 UI 1차 완성 | 2025-12-10 ~ 2025-12-12 | 사용자 단계형 워크플로 구성 | 완료 |
| M5. 보고서 해석 UX 정리 | 2025-12-13 ~ 2025-12-16 | 보고서 가독성과 신뢰도 개선 | 완료 |

| M7. 공개 준비 | 2025-12-17 ~ 2025-12-30 | 버전, 문서, 라이선스, 환경 호환성 정비 | 완료 |
| M8. 히스토리 아카이브 | 2025-12-31 ~ 2026-01-09 | 과거 프로젝트 재탐색 흐름 도입 | 완료 |

## PM 액션 아이템 관점 요약

| 우선순위 | 액션 포인트 | 해석 |
| --- | --- | --- |
| P1 | 핵심 가치 검증 | 초반에는 그래프 생성과 시뮬레이션이 실제로 이어지는지 검증하는 데 집중했다. |
| P2 | 결과물 상품화 | 중반에는 분석 결과를 리포트와 UI 흐름으로 포장해 제품 체감을 만드는 데 집중했다. |
| P3 | 재사용성과 잔존성 강화 | 후반에는 히스토리, README, 데모, Docker를 통해 재방문성과 외부 진입성을 높였다. |
| P4 | 운영 품질 보강 | 마지막에는 기능 추가보다 예외 처리, 렌더링, 인코딩, 성능 같은 실전 품질 이슈 대응이 우선이었다. |

## 발표용 짧은 버전

> MiroFish의 개발은 그래프 생성 MVP에서 시작해 시뮬레이션 엔진과 자동 리포트 기능을 결합하는 방향으로 확장됐고, 이후 단계형 UI, 대화형 분석, 히스토리 아카이브, 배포 및 안정화 작업 순으로 제품화를 진행한 것으로 정리할 수 있다.
