# MiroFish 추정 개발 로드맵

`commit_archieve`의 219개 커밋을 바탕으로 역으로 추정한 기획 문서용 로드맵이다.
이 문서는 실제 작성자의 공식 계획 문서가 아니라, 커밋 흐름과 변경 범위를 기반으로 재구성한 제품 로드맵 요약이다.

## 한 줄 요약

MiroFish는 `문서/텍스트를 그래프로 구조화하는 엔진`에서 출발해, `멀티에이전트 시뮬레이션`, `자동 리포트 생성`, `후속 대화형 분석`, `히스토리 탐색`, `배포 및 운영 안정화` 순으로 확장된 것으로 보인다.

## 기획 관점 해석

| 관점 | 해석 |
| --- | --- |
| 제품 출발점 | 단순 채팅 앱이 아니라, 텍스트를 구조화하고 그 위에서 집단 지능 시뮬레이션을 수행하는 분석 엔진으로 시작했다. |
| 중간 전환점 | 2025-12-09의 Report Agent 도입이 가장 큰 전환점이다. 이 시점부터 프로젝트가 그래프 도구에서 "예측/분석 제품"으로 성격이 바뀐다. |
| 제품화 전환 | 2025-12-10 이후 프론트 스텝 UI가 붙으며 내부 기능 묶음이 실제 사용자 흐름으로 재구성됐다. |
| 차별화 포인트 | 단순 리포트 생성에 멈추지 않고, `Step5Interaction`과 `HistoryDatabase`를 통해 결과 재해석과 축적 자산 탐색까지 확장했다. |
| 후반부 우선순위 | 신규 기능보다 배포, 인코딩, 대용량 그래프 조회, reasoning model 출력 예외 대응 등 운영 품질 개선에 무게가 실렸다. |

## 기획 문서에 바로 넣기 좋은 버전

> MiroFish는 지식 그래프 생성 기술을 기반으로 시작해, 멀티에이전트 시뮬레이션과 자동 리포트 생성 기능을 결합한 뒤, 단계형 UI와 후속 대화 기능을 더해 분석형 제품으로 발전했다. 이후 히스토리 아카이브와 배포 체계를 추가하고, 마지막 구간에서는 report agent 품질, 그래프 성능, reasoning model 호환성 같은 운영 안정화 작업에 집중한 것으로 해석된다.
